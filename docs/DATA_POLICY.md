# Data Policy

## Ownership

Data generated by this platform belongs to the researchers who produced it. The platform template itself is open-source (MIT); the data produced using it is not automatically open.

Researchers using this platform in an institutional setting should verify that their data ownership arrangements are compatible with their institution's IP and tech transfer policies.

## Privacy

- The platform does not collect personal data by default.
- Manifests record machine hostnames and runner labels. If this is sensitive, configure `RUNNER_NAME` to use a pseudonym.
- Published data is accessible to anyone with access to the dashboard. Do not publish data containing personal or sensitive information without appropriate review.

## Sharing

- Curated data with `visibility: private` is accessible only to repository collaborators.
- Published data with `visibility: public` is served via GitHub Pages and is fully open.
- The platform does not currently support fine-grained access control within the `group` visibility class. Use GitHub repository permissions for group-level access.

## Deletion

- Ephemeral data is garbage-collected according to the schedule in [`RETENTION.md`](RETENTION.md).
- Curated data can be deleted by the repository owner at any time.
- Published data follows the immutability rules in [`GOVERNANCE.md`](GOVERNANCE.md): superseded, not deleted.
